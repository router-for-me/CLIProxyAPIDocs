# Gemini CLI

Запустите сервер CLIProxyAPI и выберите одну из следующих конфигураций в зависимости от вашего метода аутентификации.

## Вход через Google (OAuth)

Установите переменную окружения CODE_ASSIST_ENDPOINT на URL-адрес сервера CLI Proxy API.

```bash
export CODE_ASSIST_ENDPOINT="http://127.0.0.1:8317"
```

Сервер будет ретранслировать запросы `loadCodeAssist`, `onboardUser` и `countTokens`. А также автоматически распределять нагрузку запросов на генерацию текста между несколькими аккаунтами.

> [!NOTE]  
> Эта функция разрешает только локальный доступ, так как в данный момент нет способа аутентификации запросов.   
> 127.0.0.1 жестко закодирован для балансировки нагрузки.

## Использование Gemini API Key

Настройте базовый URL Gemini API и API-ключ:

```bash
export GOOGLE_GEMINI_BASE_URL="http://127.0.0.1:8317"
export GEMINI_API_KEY="sk-dummy"
```

> [!NOTE]  
> В отличие от режима OAuth, этот режим позволяет настроить CLIProxyAPI на любой IP-адрес или домен.
