# Z.AI / GLM (вход через OAuth)

CLIProxyAPI поддерживает тарифы Z.AI GLM для кодинга через OAuth — без API-ключа. Доступны два провайдера идентификации: **Z.AI международный** (`chat.z.ai`) и **BigModel / Zhipu** (материковый Китай, `bigmodel.cn`). После входа аккаунт доступен как провайдер `zai` и по умолчанию обслуживает модели GLM через Anthropic-совместимую конечную точку (`https://api.z.ai/api/anthropic` для Z.AI, `https://open.bigmodel.cn/api/anthropic` для BigModel).

## Вход

### Z.AI (международный)

```bash
./cli-proxy-api --zai-login
```

### BigModel (материковый Китай)

```bash
./cli-proxy-api --zai-login --zai-provider bigmodel
```

API-ключ не нужен — после авторизации вход автоматически выпускает стандартный API-ключ тарифа (тот же шаг, что выполняет официальный клиент).

Опции: добавьте `--no-browser`, чтобы вывести ссылку для входа вместо открытия браузера.

- **Z.AI** использует серверный CLI-поток (на основе опроса); локальный сервер обратного вызова не запускается.
- **BigModel** завершает вход через локальный loopback-callback. По умолчанию занимает автоматический свободный порт; переопределите его через `--oauth-callback-port <port>`:

```bash
./cli-proxy-api --zai-login --zai-provider bigmodel --oauth-callback-port <port>
```

Поскольку callback BigModel — это loopback-адрес, браузер и прокси должны работать на одном хосте (для CLI-входа это всегда так).

## Поддерживаемые модели

- `glm-4.5`
- `glm-4.5-air`
- `glm-4.6`
- `glm-4.7`
- `glm-5`
- `glm-5-turbo`
- `glm-5.1`
- `glm-5.2`

Модели GLM — гибридные, с рассуждением: рассуждение можно отключить, а `glm-5.2` предоставляет два уровня усилия рассуждения (`high`, `max`).

## Управление моделями

Используйте канал `zai` в `oauth-model-alias`, чтобы показать модель под другим именем, видимым клиенту:

```yaml
oauth-model-alias:
  zai:
    - name: "glm-4.6"
      alias: "glm-latest"
```

Используйте тот же канал в `oauth-excluded-models`, чтобы скрыть модели из списка и маршрутизации:

```yaml
oauth-excluded-models:
  zai:
    - "glm-4.5-air"
```

## Примечания к запросам

Токен OAuth сам по себе не является учётными данными для инференса, поэтому вход выпускает стандартный API-ключ тарифа и отправляет его как `x-api-key` на Anthropic-совместимую конечную точку (без проверки клиента/капчи). Выпущенный ключ сохраняется в файлах аутентификации и используется для маршрутизации моделей; запросы GLM транслируются через Anthropic-совместимый путь.
